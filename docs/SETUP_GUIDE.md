## ЧАСТЬ 3. ПОШАГОВАЯ НАСТРОЙКА

### Шаг 1 — Первый запуск системы

```
1. Запустить Docker: docker compose up -d
2. Запустить backend: npm run start:dev
3. Создать супер-админа:
   node scripts/make-super-admin.js admin@example.com
4. Открыть: http://localhost:3000/api/admin
5. Войти: email + пароль
```

### Шаг 2 — Создать структуру объекта

Порядок создания строго сверху вниз (каждый следующий уровень ссылается на предыдущий):

#### 2.1 Организация

В админ-панели: **Организации → Создать**

```
Поле              Пример значения
───────────────   ────────────────────────────
name              ООО "Управляющая компания"
description       УК по Ленинскому р-ну
```

В базе создаётся:
```sql
INSERT INTO organizations (name, description)
VALUES ('ООО "УК"', 'Описание');
-- Получаем id: "a1b2c3d4-..."
```

#### 2.2 Жилой комплекс

**ЖК → Создать** → выбрать организацию из списка

```
Поле              Пример значения
───────────────   ────────────────────────────
name              ЖК "Солнечный"
organization_id   [выбрать из списка]
address           г. Москва, ул. Ленина, 5
```

#### 2.3 Здание

**Здания → Создать** → выбрать ЖК

```
Поле              Пример значения
───────────────   ────────────────────────────
name              Корпус 1
complex_id        [выбрать ЖК из списка]
address           ул. Ленина, 5 корп.1
```

### Шаг 3 — Добавить квартиры

#### Вариант A: Одна квартира

**Квартиры → Создать**:
```
number:    101
floor:     1
extension: 101   ← номер для вызова с домофона (если SIP)
building:  [Корпус 1]
```

#### Вариант B: Массовый импорт (CSV)

Создать файл `apartments.csv`:
```csv
number,floor,extension
101,1,101
102,1,102
103,1,103
201,2,201
202,2,202
301,3,301
```

Загрузить через API:
```bash
curl -X POST http://localhost:3000/api/buildings/1/apartments/import \
  -H "Authorization: Bearer ВАШ_ТОКЕН" \
  -F "file=@apartments.csv"
```

```
URL: http://ВАШ_СЕРВЕР:3000/api/webhooks/akuvox
```


#### 4.3 Видеорегистратор Uniview (NVR)

NVR — это устройство с несколькими камерами. Каждую камеру добавляете отдельным устройством с разным `default_channel`:

```
Устройство 1: NVR - Камера входа
  host: 192.168.1.52, type: UNIVIEW_NVR, channel: 1

Устройство 2: NVR - Камера парковки
  host: 192.168.1.52, type: UNIVIEW_NVR, channel: 2

Устройство 3: NVR - Камера двора
  host: 192.168.1.52, type: UNIVIEW_NVR, channel: 3
```

Все три устройства смотрят на один и тот же IP (NVR), но разные каналы.

### Шаг 5 — Зарегистрировать жильцов

#### 5.1 Создать пользователя

Через Swagger (`POST /api/auth/register`) или API:

```json
{
  "email": "ivanov@mail.ru",
  "phone": "+79001234567",
  "password": "Пароль123",
  "name": "Иван Иванов"
}
```

#### 5.2 Привязать к квартире

```bash
# Узнать ID пользователя
GET /api/users?email=ivanov@mail.ru

# Привязать к квартире
POST /api/users/{userId}/apartments
{
  "apartmentId": 101
}
```

Теперь Иванов видит в приложении устройства своего корпуса и получает уведомления о звонке в домофон.

#### 5.3 Заявка от жильца (альтернатива)

Жилец может сам подать заявку через приложение:
```
Приложение → Квартиры → Подать заявку → номер квартиры
```
Администратор видит заявку и одобряет/отклоняет.

### Шаг 6 — Синхронизация контактов на панель Akuvox

После добавления жильцов — синхронизировать список на домофон:

```bash
POST /api/panel-residents/{deviceId}/sync
Authorization: Bearer ВАШ_ТОКЕН
```

Система заливает на панель список имён и ID, привязывает этажи/реле.

Статусы синхронизации (`panel_residents.sync_status`):
```
synced         ← Успешно синхронизирован
pending_add    ← Ожидает добавления на панель
pending_update ← Ожидает обновления
pending_delete ← Ожидает удаления
error          ← Ошибка, см. sync_error
```

### Что видит каждая роль

```
┌──────────────────┬──────────────────────────────────────────────────────┐
│     Роль         │  Что видит / может делать                            │
├──────────────────┼──────────────────────────────────────────────────────┤
│ SUPER_ADMIN      │ ВСЁ: все организации, ЖК, здания, пользователи       │
│                  │ Может создавать ORG_ADMIN, менять роли               │
├──────────────────┼──────────────────────────────────────────────────────┤
│ ORG_ADMIN        │ Свою организацию + все ЖК и здания внутри            │
│                  │ Создаёт COMPLEX_MANAGER для своей орг                │
│                  │ Привязан к organization_id                           │
├──────────────────┼──────────────────────────────────────────────────────┤
│ COMPLEX_MANAGER  │ Свой ЖК + все здания внутри                         │
│                  │ Управляет квартирами, устройствами, жильцами         │
│                  │ Привязан к complex_id                                │
├──────────────────┼──────────────────────────────────────────────────────┤
│ RESIDENT         │ Только свои здания (через user_apartments)           │
│                  │ Видеопоток, открытие двери, события                  │
│                  │ НЕ видит других жильцов и другие здания              │
└──────────────────┴──────────────────────────────────────────────────────┘
```

### Как назначить роль

```bash
# Сделать SUPER_ADMIN:
node scripts/make-super-admin.js admin@example.com

# Назначить ORG_ADMIN (через Swagger):
PATCH /api/users/{id}
{
  "role": "ORG_ADMIN",
  "organizationId": "uuid-организации"
}

# Назначить COMPLEX_MANAGER:
PATCH /api/users/{id}
{
  "role": "COMPLEX_MANAGER",
  "complexId": "uuid-жк"
}
```

## ЧАСТЬ 5. СОБЫТИЯ И УВЕДОМЛЕНИЯ

### Типы событий в системе

```
┌──────────────────┬─────────────────────────────────────────────────────┐
│  event_type      │  Когда создаётся                                    │
├──────────────────┼─────────────────────────────────────────────────────┤
│ incoming_call    │ Нажали кнопку вызова на домофоне Akuvox              │
│ door_open        │ Дверь открыта (из приложения или вебхука)            │
│ motion           │ Uniview обнаружил движение (через WebSocket)         │
│ alarm            │ Тревога / сработал датчик                           │
│ tamper           │ Попытка вскрыть устройство                          │
│ doorbell         │ Uniview звонок в дверь (DoorBell LiteAPI event)     │
│ dial             │ Исходящий звонок с панели (SIP)                     │
│ user_add         │ Синхронизирован новый контакт на панель              │
└──────────────────┴─────────────────────────────────────────────────────┘
```

### Как работают push-уведомления

```
Для получения push на телефон нужно:

1. Firebase проект + google-services.json (Android)
   или GoogleService-Info.plist (iOS)

2. В .env указать:
   FCM_SERVICE_ACCOUNT_JSON={"type":"service_account",...}
   или
   GOOGLE_APPLICATION_CREDENTIALS=./serviceAccountKey.json

3. Приложение при логине отправляет push_token:
   POST /api/users/push-token
   { "token": "fcm-device-token", "platform": "android" }

4. При входящем звонке → backend отправляет FCM Push
   → телефон показывает уведомление даже если приложение закрыто
```

## ЧАСТЬ 7. СПРАВОЧНИК API (часто используемые)

### Аутентификация
```
POST /api/auth/login          → {accessToken, refreshToken, user}
POST /api/auth/register       → создать аккаунт
POST /api/auth/refresh        → обновить токен
POST /api/auth/logout         → выход
```

### Структура объектов
```
GET  /api/organizations                    → список УК
POST /api/organizations                    → создать УК
GET  /api/residential-complexes            → список ЖК
POST /api/residential-complexes            → создать ЖК
GET  /api/buildings                        → список зданий
POST /api/buildings                        → создать здание
GET  /api/apartments?buildingId=1          → квартиры в здании
POST /api/buildings/1/apartments/import    → импорт CSV
```

### Устройства
```
GET  /api/devices                          → список устройств
POST /api/buildings/1/devices              → добавить устройство
GET  /api/devices/{id}                     → инфо об устройстве
POST /api/devices/{id}/test-connection     → проверить связь
POST /api/devices/{id}/open-door           → открыть дверь
GET  /api/devices/{id}/live-url            → RTSP/HLS URL видео
GET  /api/devices/{id}/events              → события устройства
```

### Uniview специфика
```
POST /api/devices/{id}/ws/start            → запустить WebSocket событий
POST /api/devices/{id}/ws/stop             → остановить
GET  /api/devices/{id}/channels            → список каналов NVR
GET  /api/devices/{id}/recordings          → архив записей
GET  /api/devices/{id}/playback-url        → URL для воспроизведения
GET  /api/devices/{id}/ptz/capabilities    → поддерживает ли PTZ
POST /api/devices/{id}/ptz/move            → повернуть камеру
POST /api/devices/{id}/ptz/stop            → остановить PTZ
```

### Жители и квартиры
```
GET  /api/users                            → список пользователей
POST /api/users/{id}/apartments            → привязать к квартире
GET  /api/panel-residents/{deviceId}       → контакты на панели Akuvox
POST /api/panel-residents/{deviceId}/sync  → синхронизировать на панель
```

## ЧАСТЬ 8. ЧЕКЛИСТ ПРИ ЗАПУСКЕ НОВОГО ОБЪЕКТА

```
□ 1. Создать организацию
□ 2. Создать ЖК
□ 3. Создать здание(я)
□ 4. Импортировать квартиры (CSV или по одной)
□ 5. Добавить домофон(ы) — тип AKUVOX
     □ Проверить подключение (/test-connection)
     □ Настроить вебхук на самой панели → /api/webhooks/akuvox
     □ Указать MAC-адрес панели в системе
□ 6. Добавить камеры — тип UNIVIEW_IPC или UNIVIEW_NVR
     □ Проверить подключение
     □ Запустить WebSocket для событий (/ws/start)
□ 7. Создать аккаунты для жильцов
     □ Привязать к квартирам
□ 8. Синхронизировать контакты на панели (/sync)
□ 9. Настроить push-уведомления (Firebase)
□ 10. Протестировать полный цикл:
     □ Нажать кнопку домофона → пришёл push на телефон
     □ Из приложения нажать "Открыть" → дверь открылась
     □ Открыть видео → показывается поток
```
