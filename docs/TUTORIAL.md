# Часть 1 — Оборудование и сеть

## 1. Что нужно купить

### Минимальный комплект на один подъезд:

| Оборудование | Назначение | Примерная цена |
|---|---|---|
| Uniview NVR (8 каналов) | Запись видео 24/7 | 15 000 — 25 000 ₽ |
| Uniview IP-камеры (2-8 шт) | Видеонаблюдение по этажам | 3 000 — 8 000 ₽/шт |
| Uniview вызывная панель | Домофон у подъезда | 15 000 — 40 000 ₽ |
| PoE-коммутатор (8-16 портов) | Питание камер по кабелю | 3 000 — 8 000 ₽ |
| Мини-ПК (Intel N100 или аналог) | Запуск go2rtc, VPN | 5 000 — 10 000 ₽ |
| Роутер с SIM-картой | Интернет в здании | 3 000 — 6 000 ₽ |
| Кабель UTP Cat5e/6 | Подключение камер | по длине |

> **NVR** — это видеорегистратор. Он хранит видео с камер и отдаёт живой поток.
> **PoE** — питание по Ethernet кабелю. Камеры не нужно подключать к розетке отдельно.

## 2. Физическое подключение

### Схема подключения:

```
[Камера 1] ──┐
[Камера 2] ──┤
[Камера 3] ──┼──► [PoE коммутатор] ──► [NVR Uniview]
[Камера N] ──┘                               │
                                             │ LAN
[Вызывная панель] ───────────────────────────┤
[Мини-ПК (go2rtc)] ──────────────────────────┤
                                             │
                                      [Роутер/интернет]
                                             │
                                         Интернет
                                             │
                                    [Сервер GRG Mobile]
```

### Порядок подключения:
1. Подключи PoE-коммутатор к розетке
2. Подключи камеры к PoE-коммутатору (камеры получат питание и сеть)
3. Подключи NVR к PoE-коммутатору или отдельному порту роутера
4. Подключи вызывную панель к роутеру или коммутатору
5. Подключи мини-ПК к роутеру
6. Роутер — в интернет (SIM или кабель провайдера)

### Назначь статические IP-адреса устройствам

В настройках роутера назначь фиксированные IP каждому устройству по MAC-адресу:

| Устройство | IP-адрес (пример) |
|---|---|
| NVR | `192.168.1.100` |
| Вызывная панель | `192.168.1.50` |
| Мини-ПК | `192.168.1.10` |
| Камеры | `192.168.1.101`, `192.168.1.102`... (назначает NVR) |

> Статические IP нужны чтобы адреса не менялись после перезагрузки.

## 3. Первоначальная настройка NVR

### Зайди в веб-интерфейс NVR

Подключись к NVR через браузер (с компьютера в той же сети):
```
http://192.168.1.100
```
Логин: `admin`, пароль: заводской (обычно `Admin123` или на наклейке).

**При первом входе система попросит сменить пароль — сделай это и запомни.**

### Проверь сетевые настройки NVR

`Конфигурация → Сеть → Основные настройки`:
- IP-адрес: `192.168.1.100` (статический)
- Маска: `255.255.255.0`
- Шлюз: IP роутера (обычно `192.168.1.1`)
- DNS: `8.8.8.8`

### Проверь RTSP порт

`Конфигурация → Сеть → Расширенные настройки → Доступ к устройству`:
- RTSP порт: `554` (по умолчанию)
- HTTP порт: `80` (по умолчанию)

Запомни эти порты — они понадобятся при добавлении в систему.

| Канал 1 | Вход/лобби (этаж 0 или общий) |
| Канал 2 | Коридор 1-й этаж |
| Канал 3 | Коридор 2-й этаж |
| Канал 4 | Коридор 3-й этаж |
| ... | ... |

Это нужно для настройки видимости по этажам в системе GRG.

## 5. Первоначальная настройка вызывной панели

Логин: `admin`, пароль: заводской.

`Конфигурация → Сеть`:

- IP: `192.168.1.50`
- Логин: `admin`
- Пароль: тот что задал
- HTTP порт: `80`
- RTSP порт: `554`
- Канал: `1`

## 6. Настройка мини-ПК в здании

Мини-ПК — это маленький компьютер который стоит в серверном шкафу здания. Он выполняет две задачи:
1. **go2rtc** — конвертирует видео с NVR в HLS для телефонов через интернет
2. **WireGuard** — VPN-туннель для связи с центральным сервером

### Установи Ubuntu (или любой Linux)

Рекомендуется Ubuntu 22.04 Server.

### Установи go2rtc

```bash
wget https://github.com/AlexxIT/go2rtc/releases/latest/download/go2rtc_linux_amd64
chmod +x go2rtc_linux_amd64
sudo mv go2rtc_linux_amd64 /usr/local/bin/go2rtc
```

Создай конфиг `/etc/go2rtc.yaml`:
```yaml
api:
  listen: :1984
rtsp:
  listen: :8554
log:
  level: info
```

Запусти как сервис:
```bash
sudo nano /etc/systemd/system/go2rtc.service
```
```ini
[Unit]
Description=go2rtc
After=network.target

[Service]
ExecStart=/usr/local/bin/go2rtc -config /etc/go2rtc.yaml
Restart=always

[Install]
WantedBy=multi-user.target
```
```bash
sudo systemctl enable go2rtc
sudo systemctl start go2rtc
```

### Установи WireGuard и подключись к серверу

```bash
sudo apt install wireguard
```

Конфиг туннеля получишь у администратора центрального сервера (файл `wg0.conf`).
```bash
sudo cp wg0.conf /etc/wireguard/
sudo systemctl enable wg-quick@wg0
sudo systemctl start wg-quick@wg0
```

После этого мини-ПК получит VPN-адрес, например `10.0.1.2`.

### Проверь связь с NVR

```bash
ping 192.168.1.100   # NVR должен отвечать
curl http://192.168.1.100  # должна вернуться страница NVR
```

## 8. Создание структуры

Система работает по иерархии:
```
Организация (УК)
  └── ЖК (жилой комплекс)
        └── Здание (корпус)
              ├── Квартиры
              └── Устройства (панель, камеры)
```

### Создай Организацию

Вкладка **Организации** → форма:
- Название: `ООО УК Центральная`
- Нажми **Создать**

### Создай ЖК

Вкладка **ЖК** → форма:
- Название: `ЖК Солнечный`
- Организация: выбери из списка
- Нажми **Создать**

### Создай Здание

Вкладка **Здания** → форма:
- Название: `Корпус 1`
- ЖК: выбери из списка
- Нажми **Создать**

- Здание: выбери

## 10. Добавление устройств в систему

### Добавление вызывной панели

Вкладка **Устройства** → форма внизу страницы:

| Поле | Значение |
|---|---|
| Здание | выбери из списка |
| Имя | `Панель подъезд №1` |
| Host (IP) | `192.168.1.50` (IP панели) — или VPN-адрес если сервер на VPS |
| Тип | `UNIVIEW_IPC` |
| Роль | `Домофон` |
| Логин | `admin` |
| Пароль | пароль панели |
| HTTP порт | `80` |
| RTSP порт | `554` |
| Канал | `1` |
| Этаж | **оставь пустым** (панель видят ВСЕ жители) |

Нажми **Добавить**.

Затем нажми **"Проверить связь"** — должно появиться `Устройство доступно`.

> **Важно про IP:** Если твой сервер на VPS (в интернете), а устройства в локальной сети здания — используй **VPN-адреса** устройств (например `10.0.1.50`), а не локальные `192.168.1.x`. Локальные адреса видны только внутри здания.

### Добавление камер NVR (быстрый способ)

На вкладке **Устройства** прокрути вниз до раздела **"Быстрое добавление каналов NVR"**:

| Поле | Значение |
|---|---|
| Здание | выбери |
| IP NVR | `192.168.1.100` (или VPN-адрес) |
| Логин | `admin` |
| Пароль | пароль NVR |
| HTTP порт | `80` |
| RTSP порт | `554` |
| Канал от | `1` |
| Канал до | `8` (сколько камер подключено) |
| Префикс имени | `Камера` |

Нажми **Создать каналы** — появятся устройства: `Камера 1`, `Камера 2`, ... , `Камера 8`.

### Задай этажи камерам

Нажми **Изменить** у каждой камеры и заполни поле **Этаж**:

| Камера (канал NVR) | Где висит | Поле "Этаж" |
|---|---|---|
| Камера 1 (канал 1) | Вход/лобби | **пусто** — видят все |
| Камера 2 (канал 2) | Коридор 1-й этаж | `1` |
| Камера 3 (канал 3) | Коридор 2-й этаж | `2` |
| Камера 4 (канал 4) | Коридор 3-й этаж | `3` |
| Камера 5 (канал 5) | Паркинг | **пусто** — видят все |

> **Логика:** Этаж пустой = камеру видят все жители здания. Этаж = `2` = камеру видят только жители 2-го этажа.

## 11. Настройка уведомлений о звонке

Чтобы приложение получало уведомление когда гость звонит в домофон, нужно настроить HTTP-уведомления прямо на панели Uniview.

### Зайди в веб-интерфейс панели

```
http://192.168.1.50
```

### Найди раздел HTTP-уведомлений

Раздел может называться по-разному в зависимости от прошивки:
- `Конфигурация → Сеть → Расширенные настройки → HTTP прослушивание`
- или `Событие → Уведомление → HTTP`
- или `Configuration → Network → Advanced → HTTP Listening`

### Укажи адрес сервера GRG

```
URL: http://[VPN-адрес мини-ПК]:3000/api/webhooks/uniview
Метод: POST
```

Или если сервер имеет прямой IP:
```
URL: http://[IP-сервера]:3000/api/webhooks/uniview
```

Заголовок безопасности:
```
Имя заголовка: X-Webhook-Secret
Значение: [значение WEBHOOK_SECRET из файла .env на сервере]
```

### Включи события

Поставь галочки:
- ✅ **DoorBell** / Вызов / Call (нажатие кнопки вызова)
- ✅ **Door Open** / Открытие двери

### Сохрани и проверь

Нажми кнопку вызова на панели. В логах бэкенда должна появиться строка:
```
[WebhooksController] Uniview event: DoorBell, device: ...
```

## 16. Полный тест

### Тест 1 — Вход и просмотр устройств

1. Открой приложение
2. Войди как житель: `ivanov@example.com` / `Пароль123!`
3. На главном экране должны появиться:
   - Панель подъезда
   - Камера лобби
   - Камера коридора 1-го этажа
   - **Камеры других этажей не должны быть видны**

### Тест 2 — Просмотр видео с камеры

1. Нажми на камеру
2. Должно открыться живое видео
3. Задержка в норме: WiFi ~0.5 сек, мобильный интернет ~2-4 сек

**Что происходит под капотом:**
```
Нажал камеру
    ↓
Приложение → сервер → "дай ссылку на видео камеры 2"
    ↓
Сервер → NVR LiteAPI → получает RTSP URL
Сервер → go2rtc (мини-ПК) → регистрирует поток
Сервер → отвечает приложению: {rtspUrl, hlsUrl}
    ↓
WiFi:   приложение играет RTSP напрямую с NVR
LTE:    приложение играет HLS с go2rtc (мини-ПК → интернет)
```

### Тест 3 — Звонок в домофон

1. Нажми кнопку вызова на панели Uniview
2. На телефоне жителя должно появиться push-уведомление `"Вам звонят"` (требует Firebase)
3. Нажми на уведомление → откроется экран с видео с панели
4. Нажми **"Открыть дверь"** → замок откроется

### Тест 4 — Открытие двери из приложения

1. На главном экране нажми на панель подъезда
2. Нажми **"Открыть дверь"**
3. Слышишь щелчок реле → дверь открыта
4. В журнале событий на сервере должна появиться запись

## Частые проблемы

| Проблема | Причина | Решение |
|---|---|---|
| "Устройство недоступно" | Неверный IP или порт | Проверь IP, пинг с сервера до устройства |
| Видео не открывается в браузере | RTSP не работает в браузере | Проверь что go2rtc запущен на мини-ПК |
| Видео не идёт через интернет | go2rtc не доступен снаружи | Проверь `GO2RTC_PUBLIC_URL` в .env сервера |
| Житель не видит камеры | Нет привязки к квартире | Привяжи пользователя к квартире |
| Push-уведомления не приходят | Нет Firebase | Настрой Firebase, добавь ключи в .env |
| Неверный логин/пароль | Пароль устройства не тот | Зайди в веб-интерфейс устройства и проверь |
| Камера чёрный экран в NVR | Нет питания или кабель | Проверь PoE питание, обожми кабель |
